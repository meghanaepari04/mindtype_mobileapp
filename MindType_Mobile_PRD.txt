2. Executive Summary
MindType Mobile is an Android application that passively monitors a user's mental stress state by analyzing how they type on their smartphone — specifically the timing patterns of keystrokes — rather than what they type. It is the mobile extension of the MindType desktop application, which has already been built and deployed for Windows.

The application operates as a custom Android keyboard (Input Method Editor) that runs silently in the background during normal phone usage. It captures keystroke timing metadata including dwell time, flight time, typing speed, touch pressure, touch area, and device gyroscope readings. This data is processed locally on the device using a TensorFlow Lite machine learning model to classify the user's stress level in real time as Calm, Mild Stress, or High Stress.

The system is designed for an academic research context. Its primary goal is to collect a labeled dataset of keystroke features paired with self-reported stress scores, which will be used to train and evaluate machine learning models. A secondary goal is to provide real-time stress awareness feedback to participants during the data collection period.

Privacy is a core design principle. No typed text content is ever recorded or stored. Only timing metadata in milliseconds is captured. All inference runs on-device with no data transmitted to external servers.

3. Problem Statement & Goals
3.1 Problem Statement
Mental stress among students and working professionals is widespread but rarely monitored continuously. Existing stress assessment methods — such as psychological surveys, HR check-ins, and wearable biosensors — are either infrequent, intrusive, expensive, or unsuitable for passive monitoring during daily activities.

Most people spend a significant portion of their day typing on mobile devices. Research in behavioral biometrics indicates that cognitive and emotional states directly influence motor behavior, including typing rhythm. Stress and cognitive fatigue manifest as changes in typing speed, key hold durations, inter-key gaps, and even touch pressure. These signals can be passively captured without interrupting the user.

There is currently no lightweight, privacy-preserving Android application that continuously monitors mental stress through keystroke dynamics and provides real-time feedback — while also collecting a properly labeled dataset suitable for machine learning research.

3.2 Goals
Primary Goals
•Build a functional Android IME (custom keyboard) that passively captures keystroke timing metadata without recording typed content
•Collect a labeled dataset of keystroke features paired with self-reported stress scores from 20+ participants over 2-3 days each
•Deploy a TensorFlow Lite model on-device to provide real-time stress classification
•Export a clean, structured CSV dataset for ML model training and research analysis

Secondary Goals
•Provide participants with real-time stress awareness feedback (Green/Yellow/Red indicator)
•Visualize stress trends over time in a simple dashboard
•Compare model performance between personalized (per-user) and generalized (all-user) approaches
•Validate system reliability against self-reported ground truth labels

5. Scope
5.1 In Scope
•Custom Android keyboard (IME) built using InputMethodService
•Keystroke timing capture: keyCode, downTime, eventTime, touch pressure, touch size
•Gyroscope data capture for hand tremor analysis
•Feature extraction engine: 9 behavioral features per 60-second window
•Local SQLite database using Room for all data storage

•Real-time visual feedback overlay (Green / Yellow / Red)
•Stress trend dashboard showing patterns over time

•Cloud storage or remote data sync
•Clinical or medical diagnosis — this is a research tool only
•Recording, storing, or transmitting any typed text content
•Public app store (Google Play) distribution in this phase
•Multi-language keyboard support beyond default system language
•Integration with third-party wellness apps or health platforms

9. System Architecture Overview
MindType Mobile follows a layered architecture with all processing and storage performed on-device. There are no external API calls or backend servers. The architecture consists of six layers:

#	Layer	Components
1	Input Layer	MindTypeIMEService (InputMethodService) — captures KeyEvent and MotionEvent
2	Processing Layer	FeatureExtractor (windowing + 9-feature computation), GyroscopeManager
3	Storage Layer	Room Database (5 tables), SharedPreferences (User ID, config)
4	ML Inference Layer	StressClassifier wrapping TensorFlow Lite Interpreter, model bundled in assets/
5	Feedback Layer	FeedbackOverlay (Green/Yellow/Red), StressTrendChart, WellnessTipService
6	Export Layer	DataExporter, FileProvider, Android ShareSheet for CSV delivery

9.1 Key Android Components
Component	Type	Purpose
MindTypeIMEService	InputMethodService	Core keyboard service — captures all keystroke events
FeatureExtractor	Kotlin Class	Computes 9 behavioral features per time window
AppDatabase	Room Database	SQLite wrapper for all local data storage
StressClassifier	TFLite Wrapper	Loads model, runs inference, returns StressLevel enum
StressLabelWorker	WorkManager Worker	Schedules and displays stress self-report popups
DataExporter	Kotlin Class	Queries DB, writes CSV, triggers Android ShareSheet
FeedbackOverlay	Notification / View	Persistent stress indicator in notification bar



10. Data Schema
10.1 keystroke_events Table
Column	Type	Description
event_id	INTEGER (PK)	Auto-incremented primary key
session_id	TEXT (FK)	References sessions table
user_id	TEXT (FK)	References users table
timestamp	INTEGER	Unix timestamp in milliseconds
key_code	INTEGER	Android KeyEvent key code (no character stored)
down_time	INTEGER	System time when key was pressed (ms)
event_time	INTEGER	System time when key was released (ms)
dwell_time	REAL	Computed: event_time - down_time (ms)
flight_time	REAL	Computed: this.down_time - prev.event_time (ms)
touch_pressure	REAL	MotionEvent.getPressure() value (0.0–1.0)
touch_size	REAL	MotionEvent.getSize() value
is_backspace	INTEGER	1 if KEYCODE_DEL, 0 otherwise

10.2 feature_windows Table
Column	Type	Description
window_id	INTEGER (PK)	Auto-incremented primary key
session_id	TEXT (FK)	References sessions table
window_start	INTEGER	Start timestamp of the 60-second window
window_end	INTEGER	End timestamp of the 60-second window
mean_dwell	REAL	Mean dwell time over window (ms)
std_dwell	REAL	Standard deviation of dwell time (ms)
mean_flight	REAL	Mean flight time over window (ms)
std_flight	REAL	Standard deviation of flight time (ms)
typing_speed	REAL	Keys per minute in this window
backspace_rate	REAL	Ratio of backspace events to total events
pause_count	INTEGER	Number of gaps > 2000ms in this window
mean_pressure	REAL	Mean touch pressure value
gyro_std	REAL	Std dev of gyroscope Z-axis readings
predicted_class	TEXT	TFLite output: Calm / Mild_Stress / High_Stress
stress_label	TEXT	Self-reported class (if available for this window)

12. Key User Flows
12.1 First-Time Setup Flow
1.User installs MindType APK on Android phone
2.Launch app → Welcome screen with study description appears
3.User reads privacy notice (no text stored) → taps 'I Understand'
4.User enters assigned User ID (e.g., U01) → taps 'Start'
5.App displays keyboard setup guide → user navigates to Settings
6.User enables MindType in Language & Input → Manage Keyboards
7.User sets MindType as default keyboard
8.App detects IME is active → shows 'You're all set!' confirmation
9.Data collection begins silently in background

12.2 Active Data Collection Flow (Runs continuously)
10.User types on phone in any app (WhatsApp, browser, notes, etc.)
11.MindTypeIMEService captures each key event (timing only, no text)
12.Every 60 seconds: FeatureExtractor assembles feature vector
13.StressClassifier runs TFLite inference → outputs stress class
14.Persistent notification updates with Green / Yellow / Red dot
15.Every 10 minutes: stress popup appears → user selects 1–5 rating
16.Popup auto-dismisses if ignored → session continues uninterrupted

12.3 Data Export Flow
17.After 2–3 days: user opens MindType app → taps 'Export Data'
18.DataExporter queries Room DB for all sessions, events, labels, windows
19.CSV is generated and saved to Downloads/mindtype_mobile_dataset.csv
20.Android ShareSheet appears → user shares via Gmail / Google Drive
21.Researcher receives CSV for ML model training



13. Machine Learning Requirements
13.1 Model Input / Output Specification
Attribute	Specification
Input shape	[1 x 9] float32 array (9 features per window)
Feature order	mean_dwell, std_dwell, mean_flight, std_flight, typing_speed, backspace_rate, pause_count, mean_pressure, gyro_std
Output shape	[1 x 3] float32 probability vector
Output classes	Index 0 = Calm, Index 1 = Mild_Stress, Index 2 = High_Stress
Model format	TensorFlow Lite (.tflite), quantized for mobile
Model file location	app/src/main/assets/mindtype_model.tflite
Inference trigger	Every 60 seconds after FeatureExtractor completes
Normalization	Features normalized using min-max scaler trained on dataset

13.2 Baseline Models to Evaluate
Model	Type	Notes
Logistic Regression	Baseline	Interpretable; establishes lower-bound accuracy
SVM (RBF Kernel)	Baseline	Good for small datasets; robust to outliers
Random Forest	Ensemble	Feature importance analysis; handles noise well
XGBoost	Ensemble	Typically best on tabular data; expected top performer
LSTM	Deep Learning	Captures temporal patterns; requires more data

13.3 Evaluation Metrics
•Accuracy — overall correct predictions / total predictions
•Precision (per class) — true positives / (true positives + false positives)
•Recall (per class) — true positives / (true positives + false negatives)
•F1-Score (weighted) — harmonic mean of precision and recall
•Correlation — Pearson correlation between predicted class and self-reported score
•Confusion Matrix — to visualize class-level prediction errors
